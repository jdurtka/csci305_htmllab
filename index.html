Text Editor: Sublime

From previous experience, I knew I had no interest in either emacs or vim. I have used Notepad++ in the past, and found it quite adequate for most of my needs (primarily, a basic text editor which can handle either type of line ending), but for the sake of self-improvement I opted to look at one of the editors listed in the lab. Atom didn't seem very appealing; Sublime did.

I was able to locate a tutorial that looked promising, starting here. This is actually a tutorial in four parts, and delves into many of the useful features this editor offers, devoting the entire last two sections to some of the plugins available. Naturally, this entire HTML document was generated using Sublime (Emmet makes some parts easier, but just having multiple cursors is incredibly useful when typing repeated <li> tags.)

For the sake of brevity, the list of topics covered in the tutorial is included as an appendix.

Final Thoughts

What is the next concept that you would like to learn about your text editor?

Sublime seems to be especially powerful in its extensive use of plugins. I'm interested in finding the plugins which are most suitable to the type of work I do.

What about your VCS?

The tutorial seems fairly comprehensive and I can't think of any more advanced features that I'm interested in learning about. Going forward, it's more interesting to think of ways to work this into my normal workflow (I haven't used VCS in the past, but it has clear advantages.)

How many commits did you make while working on this project?

Throughout the course, 25 times total. Probably at least one more once this HTML document is completed.

How would you structure your commits differently next time.

Obviously, many of the commits in the course were trivial, for the sake of example. In reality it would make more sense to commit when making relatively substantive changes, although there is something to be said for ensuring that each commit represents a single logical change (rather than many disparate changes at once).
